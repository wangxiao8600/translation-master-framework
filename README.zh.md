# 翻译大师框架

翻译大师框架 v2.6 是一套在《系统与自由》21 语种出版流程中形成并迭代成型的结构化翻译框架。

它用于多语种出版、翻译与审稿工作，帮助作品在进入新语言时更贴合母语读者的表达习惯，同时继承原有结构、术语边界、叙事节奏、情感温度与文化力量。在《系统与自由》的多语种流程中，它被用于显著缓解直接 AI 机翻常见的生硬感，并成为提升作品翻译品质的一种有效方法。

真实案例见：[《系统与自由》](https://oathai.io/zh/system-and-freedom)。

这套方法来自王潇（Wang Xiao）《系统与自由》的多语种出版工程，并作为 OathAI Open Projects（开源项目）的一部分公开。

## 这个仓库提供什么

这个仓库提供一套可阅读、可引用、可 fork、可复用的翻译方法包。

它关注的核心不是把句子换成另一种语言，而是让一部作品在另一种语言中继续保持自身的结构身份、思想压力和表达温度。

它重点处理：

- 源结构
- 核心概念
- 术语边界
- 关键表达
- 叙事节奏
- 情感温度
- 文化力量
- 跨语种审稿连续性

目标是让作品在目标语言中原生思考，同时保住原作品的结构连续性。

## 核心方法

成熟工作公式可以写成：

```text
阅读源文。
核对参照术语。
在目标语言中思考。
保住结构。
让作品重新活过来。
```

在《系统与自由》的多语种工作流中，成熟阶段可以概括为：

```text
读中文原文。
检查英文术语。
在目标语言中原生思考。
```

这里的“重新活过来”，指的是在目标语言中恢复作品的结构、声音、情感温度和文化力度。

它不是神秘化表达，也不是把翻译变成口号；它描述的是一种可检查的结构恢复目标。

## 模式选择

翻译大师框架不应该对所有文本使用同一种强度。

完整模式主要为文学、哲学、诗性和高情绪密度文本设计。它的目标是 resurrection（重生）：让作品在新语言里重新活过来，同时保住结构、声音、节奏、情感密度与文化力量。

对于非虚构随笔、观察判断、技术评论和直接判断型文章，应使用 Light Alignment Mode（轻量对齐模式）：阅读源文，保留判断力度，在目标语言中以非虚构方式表达。

如果文本的力量来自现场感、粗粝感、打断感或直接判断，应使用 Raw Voice Mode（原声模式）：先保住口气，只做必要的目标语自然化。

使用原则：

```text
Use the lightest mode that preserves the work.
```

中文可写为：

```text
使用能保住作品的最轻模式。
```

不要默认使用最重模式。最重模式不等于最好模式。

选择顺序：

1. 先判断文本类型。
2. 再判断目标读者。
3. 再选择翻译模式。
4. 最后进入句子层面的重写和校准。

## 如何使用

建议先读框架概览，再按顺序使用模板：

1. `translation_task_card.md`
2. `terminology_map.md`

开始翻译前先选择模式：

- Full Translation Master Mode（完整翻译大师模式）
- Light Alignment Mode（轻量对齐模式）
- Raw Voice Mode（原声模式）

`schemas/` 和示例材料暂缓，需要单独审查后再决定是否公开。

如果需要快速理解方法边界，先看 `docs/release_boundary.md`。

公开模板只提供判断框架，不替代具体作品的翻译判断。
