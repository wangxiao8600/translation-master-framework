# Framework Overview

Translation Master Framework v2.6 is a structured translation framework formed and iterated through the 21-language publication process of *System and Freedom*.

It is designed for multilingual publishing, translation, and review work where literal conversion is not enough. The framework helps a work enter a new language while preserving structure, terminology boundaries, narrative rhythm, emotional temperature, cultural force, and target-language reader fit.

The framework is published as part of OathAI Open Projects.

---

## 1. Why This Framework Exists

Translation can fail even when individual sentences look accurate.

A work may lose:

- structural continuity
- conceptual pressure
- terminology boundaries
- narrative rhythm
- emotional temperature
- cultural force
- authorial voice

Translation Master Framework treats translation as structural inheritance rather than sentence replacement.

The central question is:

```text
How can a work think natively in another language without losing its original structural identity?
```

---

## 2. Origin

The framework emerged from the multilingual publication process of *System and Freedom*.

That project required more than text conversion. It required one work to remain identifiable across many language worlds while adapting to native reader expectations in each target language.

This pressure led to a reusable method for preserving:

- source structure
- core concepts
- terminology boundaries
- key phrases
- rhythm and narrative movement
- emotional intensity
- cultural resonance
- review continuity

---

## 3. Mature Working Formula

The mature working formula is:

```text
Read the source.
Check the reference terms.
Think in the target language.
Preserve the structure.
Resurrect the work.
```

For the original *System and Freedom* multilingual workflow, the mature form can be summarized as:

```text
Read Chinese.
Check English terms.
Think in the target language.
```

Here, "resurrect" means restoring structure, voice, emotional temperature, and cultural force in the target language.

This formula separates three roles:

- source language as semantic origin
- reference language as terminology and alignment support
- target language as living expression

The public framework generalizes this pattern without releasing the private production chain behind it.

---

## 4. Mode Selection

Translation Master Framework was designed primarily for literary, philosophical, poetic, and high-emotional-density translation.

Its full mode aims at resurrection: letting the work come alive again in a new language while preserving structure, voice, rhythm, emotional density, and cultural force.

This logic is powerful for poetic and literary texts. It should be applied more lightly to nonfiction judgment, observation pieces, technical commentary, and other direct prose forms.

Translation Master Framework should not be applied at the same intensity to every text.

Use the lightest mode that preserves the work.

Do not use the strongest mode by default. The strongest mode is not always the best mode.

Translation Master Framework is a mode-selection framework, not a one-size-fits-all polishing engine.

Selection order:

1. Identify the text type.
2. Identify the target reader.
3. Select the translation mode.
4. Then enter sentence-level rewriting and calibration.

Mode selection and thinking basis are separate decisions:

- `Translation Mode` decides the treatment intensity.
- `Translation Basis / Thinking Rule` decides what to read as source, what to use as reference, and how to think in the target language.

### Full Translation Master Mode

Use for:

- books
- manifestos
- philosophical texts
- poetic or literary texts
- high-emotional-density works

Goal:

```text
resurrection: preserve structure, voice, rhythm, emotional density, and cultural force
```

Risk if overused:

- over-polishing
- excessive symmetry
- loss of rough judgment
- turning direct nonfiction into literary prose

### Light Alignment Mode

Use for:

- nonfiction essays
- observation pieces
- current-affairs judgment
- technical commentary
- AI reality analysis

Goal:

```text
read the source, preserve judgment, think in target-language nonfiction
```

For nonfiction judgment essays, the goal is not to sculpt source sentences into beautiful target-language prose. The goal is to let the author's judgment continue happening inside target-language nonfiction.

Keep:

- terminology consistency
- structural continuity
- judgment force
- native target-language expression

Avoid:

- over-polishing
- excessive parallelism
- consultant-checklist voice unless explicitly intended
- philosophical elevation
- cultural-bridge overwork
- full resurrection tone

### Raw Voice Mode

Use when the author explicitly wants the target text to stay close to the original speaking voice.

Goal:

```text
preserve voice first; naturalize only as needed
```

This mode is useful when roughness, interruption, directness, or live judgment is part of the text's force.

## 5. Core Components

The public framework surface focuses on these components:

### Source Structure

Identify the work's argument, narrative movement, section logic, repeated anchors, and structural pressure.

### Terminology Boundary

Define which terms must remain stable, which may adapt by context, and which require human review.

### Key Phrases

Track phrases that carry conceptual force, emotional force, or recurring structural meaning.

### Target-Language Fit

Make the target text read naturally to native readers without flattening the source work.

### Rhythm And Voice

Review sentence movement, paragraph flow, directness, restraint, warmth, and pressure.

### Emotional Temperature

Preserve the force, restraint, warmth, anger, silence, or intensity that the source work requires.

### Cultural Force

Handle cultural references deliberately instead of defaulting to literal transfer or over-localization.

### Review Continuity

Keep human review anchored to structure, terminology, rhythm, emotion, and culture across revision passes.

---

## 6. Translation As Structural Inheritance

The framework treats translation as inheritance of structure.

Literal accuracy matters, but it is not sufficient by itself.

A structurally successful translation should:

- preserve the work's core shape
- keep key concepts inside their intended boundaries
- read naturally in the target language
- avoid direct AI machine-translation stiffness
- avoid flattening emotional force
- adapt cultural references with intention
- keep unresolved risks visible for review

The goal is not to make every language version identical.

The goal is to let each language version become native to its language while staying connected to the same underlying work.

## 8. Public / Invite-only / Private Layers

The framework uses three disclosure layers:

```text
Public for citation.
Invite-only for collaboration.
Private for production memory.
```

### Public

The public layer is for citation, canonical source attribution, basic reuse, and public boundary.

It may include:

- README
- license and notice
- citation metadata
- framework overview
- release boundary
- thin public templates
- reviewed schemas when cleared

### Invite-only

The invite-only layer is for collaboration with people who show visible contribution trace, citation discipline, and concrete collaboration intent.

It may include after separate approval:

- fuller templates
- redacted case notes
- short before / after fragments
- review checklist variants
- community feedback notes
- contributor experiments

### Private

The private layer is for production memory and archive continuity.

It remains private by default and includes:

- raw production archive
- raw human-AI logs
- full prompt chains
- complete language matrices
- language-specific execution details
- complete manuscript workflow
- unpublished book-production materials
- private review traces

---

## 9. What This Framework Does Not Claim

This framework does not:

- certify downstream translation quality
- guarantee high-quality output by itself
- replace human review
- require full-mode treatment for every text
- publish the full *System and Freedom* production workflow
- expose all production prompts or review evidence
- provide an OathAI endorsement of third-party implementations

Downstream use remains the responsibility of the user or implementing project.

---

## 10. Reuse Guidance

Start with:

1. `templates/translation_task_card.md`
2. `templates/terminology_map.md`

Use them to define the task before generating or reviewing translation output.

For AI-assisted translation, use the framework to:

- select the right translation mode
- set the source structure
- define target-language expectations
- constrain terminology drift
- identify cultural and emotional risks
- guide human review

Do not use the public templates as a replacement for human judgment.
